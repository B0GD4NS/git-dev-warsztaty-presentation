Ćwiczenie: proste flow

Komendy potrzebne do ćwiczenia
  • git init
  • git status
  • git add
  • git commit
  1. Utwórz na pulpicie katalogi cwiczenia/flow-demo
    cd ~/Desktop
    mkdir cwiczenia
    cd cwiczenia
    mkdir flow-demo
    cd flow-demo
  2. Utwórz repozytorium (git init)
  3. Stwórz plik index.txt z zawartością "Hello World"
  4. Dodaj plik do repozytorium (git add, git commit)
  5. Zmień treść pliku na "Witaj świecie"
  6. Sprawdź status repozytorium (git status)
  7. Dodaj plik do repozytorium (git add, git commit)
  8. Dodaj do pliku kolejną linijkę "jest git"
  9. Stwrórz nowy plik cars.json i wpisz "toyota"
  10. Sprawdź status repozytorium (git status)
  11. Dodaj plik cars.json do repozytorium
    (git add cars.json, git commit)
  12. Sprawdź status repozytorium (git status)
  13. "Skomituj" plik index.txt
  14. Usuń plik cars.json
  15. Sprawdź status repozytorium (git status)
  16. "Skomituj" usunięcie pliku
  17. Sprawdź status repozytorium (git status)

Ćwiczenie: poprawiane commitów

Komendy potrzebne do ćwiczenia
  • git add
  • git commit
  • Sklonowanie repozytorium do ćwiczenia
    cd ~/Desktop/cwiczenia
    git clone https://github.com/SkillsTemple/git-devWarsztaty-reset-revert correcting-commits--demo
Nasze eksperymentalne zmiany, których dokonaliśmy na branchu master niestety okazały się niedobre i musimy je skasować. Zostały one już wypchnięte na repozytorium publiczne, więc chcielibyśmy zrobić to w taki sposób, aby nikomu z zespołu nie przysporzyć kłopotu. Dodatkowo zmiany do CSS są rozstrzelone na kilka commitów i chcielibyśmy połączyć je w jeden. Te zmiany na szczęście są tylko na naszym komputerze, więc nie musimy obawiać się przesuwania branchy.

Zadanie:
Wycofać zmiany z commita 83bfe65 ("EXPERIMENTAL") z tagiem "already-pushed". Połączyć commity c42f4e2 c1ae555 6b0c679 61d2ada w jeden z lepszym opisem.

Ćwiczenie: pull requesty

Komendy potrzebne do ćwiczenia
  • git add
  • git commit
  1. Otwórz w przeglądarcę stronę projektu do którego będziesz kontrybuował github.com/tometchy/git-dev-warsztaty-presentation
  2. Kliknij fork
  3. cd ~/Desktop/cwiczenia
    git clone https://github.com/TWOJ_NICK/git-dev-warsztaty-presentation pull-requests-demo
  4. Dopisz swoje pytanie w pliku Questions-from-workshops-DZISIEJSZA_DATA.md przy swoim numerku
  5. Zacommituj swoje pytanie
  6. Push (do swojego repozytorium)
  7. Otwórz w przeglądarcę stronę projektu do którego kontrybuujsz github.com/tometchy/git-dev-warsztaty-presentation
  8. Kliknij New pull request i wybierz jako źródło brancha master ze swojego profilu
