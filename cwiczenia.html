Ćwiczenie: poprawiane commitów

  • Sklonowanie repozytorium do ćwiczenia
    cd ~/Desktop/cwiczenia
    git clone https://github.com/SkillsTemple/git-devWarsztaty-reset-revert correcting-commits--demo
    cd correcting-commits--demo
Nasze eksperymentalne zmiany, których dokonaliśmy na branchu master niestety okazały się niedobre i musimy je skasować. Zostały one już wypchnięte na repozytorium publiczne, więc chcielibyśmy zrobić to w taki sposób, aby nikomu z zespołu nie przysporzyć kłopotu. Dodatkowo zmiany do CSS są rozstrzelone na kilka commitów i chcielibyśmy połączyć je w jeden. Wyobrażamy sobie, że nie pobraliśmy tych zmian z serwera, tylko zakładamy, że stworzyliśmy je lokalnie, więc te zmiany na szczęście są tylko na naszym komputerze, więc nie musimy obawiać się przesuwania branchy. Po otworzeniu gitk, widzimy tag already-pushed, który jest sumulacją, że to już zostało wypushowane, a to co później jest tylko lokalnie.

Zadanie:
Wycofać zmiany z commita 83bfe65 ("EXPERIMENTAL") z tagiem "already-pushed". Połączyć commity c42f4e2 c1ae555 6b0c679 61d2ada w jeden z lepszym opisem.

  • git checkout
  • git merge
  • git rebase
  • Sklonowanie repozytorium do ćwiczenia
    cd ~/Desktop/cwiczenia
    git clone https://github.com/git-warsztaty/rebase-vs-merge-training rebase-and-merge-demo
    cd rebase-and-merge-demo
    git checkout feature-branch (tylko żeby ten branch się pojawił lokalnie)
    git checkout master

    • Merge output:

    podejrzyj rozwiązanie git checkout master
    git merge feature-branch
    przywrócenie brancha master do początkowej sytuacji git reset --hard origin/master

    Rebase output:

    podejrzyj rozwiązanie git checkout feature-branch
    git rebase master
    git checkout master
    "git reset --hard feature-branch" albo "git merge feature-branch"
